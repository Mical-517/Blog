# 二.管道符、重定向与环境变量

## 1.重定向

重定向分为输入重定向与输出重定向，其中输出重定向又分为标准输出，与错误输出重定向

### 1. 三个默认文件描述符

- **0 = stdin**  标准输入（键盘）
- **1 = stdout** 标准输出（屏幕）
- **2 = stderr** 标准错误（报错）

### 2. 常用符号

- `>`  等价于 `1>`  覆盖输出
- `>>` 等价于 `1>>` 追加输出
- `2>`  覆盖错误
- `2>>` 追加错误
- `&>`  覆盖**所有输出 + 错误**
- `&>>` 追加所有输出 + 错误

### 3. 经典用法

```bash
ls > file.txt        # 正确输出覆盖到文件
ls 2> err.log        # 错误单独存
ls > log.txt 2>&1    # 错误合并到输出（老式标准）
ls &> all.log        # 输出+错误都进文件（推荐）
ls 2>/dev/null       # 丢弃错误（最常用）
```

### 4. 输入重定向

- `<`  从文件读入，代替键盘

### 5. 核心记住一句话

**重定向是把 “输出 / 错误” 导向文件，不是改变命令权限；`2>/dev/null` 专门屏蔽权限报错。**

重点:**他的数据流是文本文件**，这一点要与管道符区分开，管道符的数据流是命令程序的结果

## 2.物理设备的命名规则

对与存储设备名字的错误解读

1. 设备名称的错误解读

   sda错误认为是物理插槽上的第一块内存，**正确的是他只是被内核识别的第一块存储设备**，设备名称只与内核识别顺序有关

2. 分区名称错误解读

   sda3错误认为sda有3个分区，**这只可以表示这是第3个分区**



**理解存储设备与分区的关系：**

每个磁盘设备都由扇区（512字节）构成，其中第一个扇区，主引导记录需要446字节，分区表要64个字节，结束符要2个字节；**其中分区表每记录一个分区信息要消耗16个字节**，所以理论上一个磁盘设备最多有4个主分区，但是这太少了，所以我们将其中一个主分区变为扩展分区，它存储的是指向其他分区的指针，这样我们就将其指向的分区称为逻辑分区



当我们添加一个硬件设备之后，操作系统就自动把这个硬件抽象成了一个/dev下的一个文件

## 3.文件系统与数据资料

当我们将一个磁盘设备划分好分区之后，我们就要在分区加上一个文件系统，**文件系统就是规范或者说规定了文件的存储位置，例如创建一个文件，这个文件的物理地址在哪里，删除文件如何删除，这都是文件系统的规则**

所以拿到一个磁盘存储设备：先分区后格式化文件系统，最后挂载



理解分区里的文件系统是如何管理文件的：

1. inode：存储文件的基础信息，不存储内容，内部有指针指向block
2. block：存储实际内容

## 5.添加磁盘设备



### 1.fdisk命令

fdisk命令用于创建，删除，修改磁盘分区，format disk，使用：fdisk 添加的硬盘

对与分区信息当中的字段id的意义：表示这个分区的类型，使用数字来表示，使用参数L可以查看所有的数字意义，做完之后，os会自动把这个分区抽象成/dev/sdb1设备文件，**之后就是格式化文件系统**,使用mkfs.(具体的文件系统如xfs，ext4) 设备文件(/dev/sdb1)



### 2.du命令

当设备文件成功挂载之后，可以利用这个挂载目录写入文件到硬件，使用之前可以使用**du命令**，它用于查看文件数据占用量即分区或者目录所占用的磁盘容量大小，即**du -sh 目录名称**,就是用来查看一个或者多个文件占用了多大的磁盘空间，**注意这里占用大小表示的就是物理大小（即占用磁盘大小），因为文件系统保存文件内容就是按照block保存的，所以它的占用容量>=文件实际容量**



## 6.添加交换分区

所谓交换分区就是当物理内存不够使用的时候，将物理内容中不经常被使用的数据，放到磁盘当中，空出空间来供进程使用。

过程大概与添加磁盘设备差不多，只是最后要修改一下分区的id，因为这个分区是交换分区不是普通的盘符，使用t参数，**同样不要忘记格式化文件系统以及挂载**



## 7.磁盘容量配额

目的：linux是一个多用户系统，就是说他要限制普通用户在一个分区中创建文件的数量以及容量，如果不加以限制，所有的用户可能创建大量的文件，消耗系统资源

如何实现：使用quota技术从软限制以及硬限制来两方面达成

软限制：达到限制容量或者数量，提出警告信息，但是还是可以创建

硬限制：不可以创建



如何使用quote：

quote作用域盘符也就是所谓一个·独立分区，在/etc/fstab中修改各个分区的文件系统权限就可以了，只需要加上quota，然后重启就可以了



### 1.使用xfs_quota命令

专门为xfs文件系统容量配额设计的命令

```shell
xfs_quota -x -c 'limit bsoft=3m bhard=6m isoft=3 ihard=6 tom ' /boot
```

这里解释参数bsoft/bhard，isoft/ihard

bsoft/bhard：就是前面提到关于文件系统如何管理文件先关的block块容量，也及时是文件占用的最大容量的总和

isoft/ihard：就是inode，表示限制文件系统最大使用的inode个数来限制文件数量



### 2.edquota命令

用户设置分区容量配额之后，可以使用edquota按需修改配额
