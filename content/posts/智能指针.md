# 智能指针

智能指针是 C++11 引入的核心特性，用于**自动化管理堆内存**，替代裸指针解决内存安全问题。核心知识点如下：

### 1. 核心原理：RAII（资源获取即初始化）

- 利用**栈对象的生命周期**控制**堆内存**：智能指针本身是栈对象，离开作用域时自动调用析构函数，在析构中释放堆内存。
- 本质：将 “手动管理堆内存” 的责任交给 “栈对象的自动析构”。

### 2. 三大智能指针（C++11/14）

|       类型        |                           核心特性                           |                      适用场景                      |
| :---------------: | :----------------------------------------------------------: | :------------------------------------------------: |
| `std::unique_ptr` | **独占所有权**：不可拷贝，只能移动；离开作用域自动释放堆内存 |   绝大多数场景（优先使用），管理无需共享的堆对象   |
| `std::shared_ptr` | **共享所有权**：通过引用计数实现，多个指针可指向同一对象；计数为 0 时自动释放 | 需多个指针共享同一对象的场景（如多线程、数据共享） |
|  `std::weak_ptr`  | **弱引用**：不增加引用计数，用于解决 `shared_ptr` 的循环引用问题 |    辅助 `shared_ptr`，避免循环引用导致内存泄漏     |

### 3. 核心操作（以 `unique_ptr` 为例）

- **创建**：`std::make_unique<T>()`（C++14 推荐，更安全）或 `std::unique_ptr<T> ptr(new T)`；
- **访问**：`ptr->成员`（访问对象成员）、`*ptr`（解引用获取对象）、`ptr.get()`（获取原始裸指针，慎用）；
- **修改**：`ptr.reset(new T)`（释放旧对象，指向新对象）、`ptr.release()`（释放所有权，返回裸指针，需手动管理）；
- **移动**：`std::move(ptr)`（转移所有权，原指针失效）。

### 4. 使用原则

- 优先用 `std::unique_ptr`，避免滥用 `std::shared_ptr`（减少循环引用风险）；
- 优先用 `std::make_unique`/`std::make_shared` 创建智能指针（更安全、简洁）；
- 避免混用智能指针和裸指针（防止重复释放、野指针）。

# 问题 1：为什么有智能指针？

智能指针的出现是为了解决**裸指针管理堆内存的三大致命问题**：

### （1）裸指针的痛点

#### ① 内存泄漏

裸指针需要手动 `delete` 释放堆内存，一旦忘记（如函数提前返回、异常抛出），堆内存永远无法回收，长期运行导致内存耗尽。

```c++
void test() {
    int* p = new int(10);
    if (/* 某个条件 */) return; // 提前返回，忘记delete → 内存泄漏
    delete p; // 只有正常走到这里才释放
}
```

#### ② 重复释放（双重 free）

多个裸指针指向同一堆内存时，若多次 `delete` 同一地址，会导致程序崩溃（未定义行为）。

```c++
int* p1 = new int(10);
int* p2 = p1; // 两个指针指向同一地址
delete p1;
delete p2; // 重复释放 → 程序崩溃
```

#### ③ 野指针

堆内存释放后，未将指针置 `nullptr`，后续误访问会导致程序崩溃或数据篡改。

```
int* p = new int(10);
delete p;
*p = 20; // 野指针访问 → 未定义行为
```

### （2）智能指针的解决方案：RAII

智能指针将 “堆内存管理” 绑定到**栈对象的生命周期**：

- 智能指针本身是栈对象，离开作用域（如函数返回、异常抛出）时，**自动调用析构函数**；
- 析构函数中自动执行 `delete`（或 `delete[]`），无需手动操作，从根本上避免内存泄漏、重复释放、野指针。



```c++
#include <memory>
void test() {
    std::unique_ptr<int> p = std::make_unique<int>(10);
    if (/* 某个条件 */) return; // 提前返回也没关系：p离开作用域自动析构，释放内存
    // 无需手动delete
} // 函数结束，p自动析构，释放堆内存
```

# 问题 2：为什么说 “具有移动语义” 是智能指针区别于栈区存数据的一大特点？

要理解这个问题，需先对比**栈区数据**和**智能指针管理堆数据**的核心差异，再看移动语义的作用：



### （1）栈区数据的局限性：生命周期固定，无法 “转移所有权”

栈区数据（如局部变量）的生命周期由作用域严格控制：

- 无法将栈对象的 “所有权” 转移给其他作用域（比如函数返回栈对象，实际是拷贝，原对象仍在栈上销毁）；
- 若想让数据在不同作用域间 “流动”（如函数返回大对象、存入容器），只能拷贝（开销大）或用裸指针（不安全）。

```
// 栈对象：无法转移所有权，返回时只能拷贝
std::string createString() {
    std::string s = "hello"; // 栈对象
    return s; // 拷贝返回（虽有优化，但本质是栈生命周期限制）
} // s销毁
```

### （2）智能指针的优势：结合 “栈的自动管理”+“堆的灵活性”+“移动语义”

智能指针管理的是**堆内存**，但通过移动语义可以：

- **转移堆内存的所有权**：无需拷贝堆数据，仅转移智能指针的 “控制权”，原智能指针失效；
- **保持栈的自动管理**：新的智能指针接管堆内存后，仍会在其作用域结束时自动释放，无需手动操作。

以 `unique_ptr` 为例（独占所有权，只能移动）：



```c++
#include <memory>
#include <iostream>

class Data {
public:
    Data() { std::cout << "Data构造" << std::endl; }
    ~Data() { std::cout << "Data析构" << std::endl; }
};

// 函数返回unique_ptr：转移堆内存所有权
std::unique_ptr<Data> createData() {
    auto ptr = std::make_unique<Data>(); // 栈上的unique_ptr管理堆Data
    return ptr; // 移动语义：转移堆Data的所有权给调用方，ptr失效
} // ptr离开作用域，但已无所有权，不析构Data

int main() {
    auto p = createData(); // p接管堆Data的所有权
    // ... 使用p ...
    return 0;
} // p离开作用域，自动析构Data
```

### （3）核心总结

- 栈区数据：生命周期固定，**无法转移所有权**，只能拷贝（开销大）；

- 智能指针：通过移动语义**转移堆内存所有权**（零拷贝，效率高），同时仍由栈对象自动管理生命周期（安全）。

  

  → 移动语义让智能指针既保留了 “栈自动管理” 的安全性，又获得了 “堆数据可跨作用域流动” 的灵活性，这是纯栈数据做不到的。

# 问题 3：如何深刻理解智能指针区分于普通指针？

可以从**5 个核心维度**对比，结合代码示例建立直观认知：

### （1）维度 1：所有权（最本质区别）

- **普通指针**：无 “所有权” 概念，多个指针可随意指向同一堆内存，谁都可以释放，容易混乱；

- **智能指针**：明确 “所有权” 规则：

  - `unique_ptr`：**独占**，同一时间只能有一个指针管理堆内存；

  - `shared_ptr`：**共享**，通过引用计数记录有多少指针共享；

    

    → 所有权规则从根源上避免重复释放、野指针。

  

```c++
// 普通指针：无所有权，混乱
int* p1 = new int(10);
int* p2 = p1; // 谁都可以指向，谁都可以delete
delete p1;
// delete p2; // 重复释放，崩溃

// unique_ptr：明确独占所有权
auto p3 = std::make_unique<int>(10);
// auto p4 = p3; // 编译报错！不能拷贝（独占）
auto p4 = std::move(p3); // 只能移动：p3失去所有权，p4接管
// *p3 = 20; // 错误：p3已无所有权
```

### （2）维度 2：生命周期管理（核心功能）

- **普通指针**：堆内存需**手动管理**，忘记 `delete` 就泄漏；
- **智能指针**：堆内存由**栈对象自动管理**，离开作用域自动释放，无需手动操作。



```c++
// 普通指针：手动管理，容易泄漏
void test1() {
    int* p = new int(10);
    // 忘记delete → 内存泄漏
}

// 智能指针：自动管理，绝不泄漏
void test2() {
    auto p = std::make_unique<int>(10);
} // p自动析构，释放内存
```

### （3）维度 3：安全性（避免三大问题）

- **普通指针**：容易出现内存泄漏、重复释放、野指针；

- **智能指针**：从设计上避免这些问题：

  - 自动释放 → 无内存泄漏；
  - 所有权规则 → 无重复释放；
  - 移动后原指针失效（`unique_ptr` 移动后为 `nullptr`）→ 减少野指针。

  

### （4）维度 4：操作限制（约束行为，减少错误）

- **普通指针**：操作自由（可随意拷贝、赋值、加减），但自由意味着风险；

- **智能指针**：有明确的操作限制：

  - `unique_ptr`：禁止拷贝，只能移动；

  - `shared_ptr`：可拷贝，但拷贝会增加引用计数；

    

    → 限制看似 “不自由”，实则减少错误。

  

### （5）维度 5：异常安全（应对异常更健壮）

若函数中抛出异常，普通指针可能因跳过 `delete` 导致泄漏；智能指针因栈对象自动析构，即使异常也能释放内存。





```c++
// 普通指针：异常时泄漏
void test3() {
    int* p = new int(10);
    throw std::runtime_error("异常"); // 抛出异常，跳过delete
    delete p; // 永远走不到这里
}

// 智能指针：异常时仍能释放
void test4() {
    auto p = std::make_unique<int>(10);
    throw std::runtime_error("异常"); // 抛出异常
} // p自动析构，释放内存
```

### （6）一句话总结区别

普通指针是 “**无约束的工具**”，灵活但风险高，所有责任由程序员承担；

智能指针是 “**有规则的管家**”，通过所有权和 RAII 约束行为，自动管理内存，把风险降到最低。

# 总结代码

```c++
#include <iostream>
#include <memory>
using namespace std;

class Data
{
public:
    Data()
    {
        cout << "Data 构造函数" << endl;
    }
    ~Data()
    {
        cout << "Data 析构函数" << endl;
    }
    void print()
    {
        cout << "Data内部函数" << endl;
    }

private:
};

int main()
{
    cout << "-------普通指针，不手动使用delete-------------" << endl;
    // 使用普通指针，不使用delete释放
    {
        Data *ptr = new Data();
    }
    // 创建智能指针
    {
        // c++11支持
        cout << "-------c++11智能指针创建-------------" << endl;
        unique_ptr<Data> ptr1(new Data);
        // c++14支持，更推荐，因为效率更高，更安全
        cout << "-------c++14智能指针创建-------------" << endl;
        auto ptr2 = make_unique<Data>();
    }
    // 使用只能指针
    {
        cout << "-------使用智能指针-------------" << endl;
        auto ptr = make_unique<Data>();
        cout << "使用智能指针访问指向类对象的内部函数:";
        ptr->print();
        cout << "-------使用智能指针得到普通普通指针-------------" << endl;
        auto ptr1 = ptr.get();
        cout << "现在普通指针与智能指针都指向同一块堆内存,此时如果手动释放堆内存，由于智能指针出了作用域自动清理堆内存，所以内存被释放两次" << endl;

        cout << "-------智能指针释放控制权-------------" << endl;
        ptr.release();
        if (ptr == nullptr)
            cout << "智能指针为空，现在只有普通指针指向堆内存" << endl;
        cout << "手动释放普通指针指向的堆内存" << endl;
        delete ptr1;

        cout << "-------智能指针重指向一个新的堆内存-------------" << endl;
        ptr = make_unique<Data>();
        ;
        ptr.reset(new Data());
    }
    {
        cout << "----------智能指针只能使用移动复制与构造---------" << endl;
        cout << "移动构造" << endl;
        auto ptr0 = make_unique<Data>();
        auto ptr1 = move(ptr0);
        if (ptr0 == nullptr)
            cout << "ptr0为空" << endl;
    }
    {
        cout << "复制构造" << endl;
        auto ptr0 = make_unique<Data>();
        unique_ptr<Data> ptr1;
        ptr1 = move(ptr0);
    }
    return 0;
}
```
