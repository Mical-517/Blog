移动语义是 C++11 为解决**深拷贝性能损耗**而生的核心特性：

> 把「不再使用的对象」的资源（堆内存、文件句柄等）直接 “转移” 给新对象，而非复制一份，实现**零拷贝开销**，尤其适合大对象（如长字符串、大容器、音视频数据）。

### 二、前置必懂：左值 / 右值 + 右值引用（移动语义的基石）

#### 1. 左值 vs 右值（通俗区分）

表格







| 类型 |                 核心特征                 |                             例子                             |
| :--: | :--------------------------------------: | :----------------------------------------------------------: |
| 左值 |  有名字、能取地址、可修改（除非 const）  |     `int a = 10;` 中的`a`；`string s = "hello";` 中的`s`     |
| 右值 | 无名字、临时存在、用完即销毁、不能取地址 | 字面量`10`、`"hello"`；函数返回的临时对象`func()`；`std::move(a)`后的`a` |

**一句话记忆**：左值是 “持久的”，右值是 “临时的”。

#### 2. 右值引用（`&&`）：专门绑定右值的引用

- 语法：`类型&& 变量名`

- 作用：唯一能绑定右值的引用类型，是实现移动语义的 “桥梁”。

- 关键特性：

  cpp

  

  运行

  

  

  

  

  ```
  int a = 10;
  int&& b = 10;  // ✅ 右值引用绑定右值（字面量10）
  int&& c = a;   // ❌ 错误：右值引用不能直接绑定左值
  int&& d = std::move(a); // ✅ std::move把左值a转为右值，可绑定
  ```

  

### 三、核心实现：移动构造函数 + 移动赋值运算符

#### 1. 移动构造函数（最常用）

##### （1）写法（固定模板）

cpp



运行









```
class String {
private:
    char* str;
    int size;
public:
    // 普通构造
    String(const char* src) : size(strlen(src)) {
        str = new char[size + 1];
        memcpy(str, src, size + 1);
        cout << "普通构造：" << (void*)str << endl;
    }

    // 深拷贝构造（对比用）
    String(const String& other) {
        size = other.size;
        str = new char[size + 1]; // 重新申请内存
        memcpy(str, other.str, size + 1); // 复制数据（慢）
        cout << "深拷贝构造：" << (void*)str << endl;
    }

    // 移动构造函数（核心！）
    String(String&& other) noexcept { // noexcept：告诉编译器不会抛异常（容器要求）
        // 第一步：偷资源（仅复制指针，无数据拷贝）
        this->str = other.str;
        this->size = other.size;

        // 第二步：源对象置空（避免析构时重复释放）
        other.str = nullptr;
        other.size = 0;

        cout << "移动构造：" << (void*)this->str << endl;
    }

    // 析构函数
    ~String() {
        if (str) {
            cout << "析构：释放" << (void*)str << endl;
            delete[] str;
        }
    }
};
```

##### （2）触发条件

只有当用**右值**初始化新对象时，才会触发移动构造：

cpp



运行









```
// 场景1：临时对象（右值）初始化 → 自动触发移动构造
String s1 = String("hello"); 

// 场景2：std::move把左值转为右值 → 触发移动构造
String s2("world");
String s3 = std::move(s2); // s2的资源被转移给s3，s2变为空
```

##### （3）核心逻辑

- 「偷」：把源对象（other）的资源指针直接赋值给新对象（无数据拷贝，O (1) 开销）；
- 「清空」：把源对象的指针置`nullptr`，避免析构时重复释放；
- 对比深拷贝：深拷贝是 “复制数据 + 新申请内存”（O (n) 开销），移动构造是 “只复制指针”（零开销）。

#### 2. 移动赋值运算符（补充）

用于给**已存在的对象**移动资源（区别于构造），写法类似：

cpp



运行









```
String& operator=(String&& other) noexcept {
    if (this == &other) return *this; // 防止自赋值

    // 第一步：释放当前对象的旧资源
    delete[] this->str;

    // 第二步：偷源对象的资源
    this->str = other.str;
    this->size = other.size;

    // 第三步：源对象置空
    other.str = nullptr;
    other.size = 0;

    cout << "移动赋值：" << (void*)this->str << endl;
    return *this;
}

// 使用
String s4;
s4 = std::move(s3); // 触发移动赋值
```

### 四、关键工具：std::move（必懂）

#### 1. 本质

`std::move` 不是 “移动”，而是**编译期类型转换**：把左值强制转为右值引用（`static_cast<T&&>(obj)`），无任何运行时开销。

#### 2. 作用

标记一个左值 “可以被移动”，从而触发移动构造 / 移动赋值（否则左值会触发深拷贝）。

#### 3. 注意

- `std::move` 后，源对象的资源已被转移，**不能再使用其资源**（比如调用`s2.getStr()`），但可以析构（因为指针已置空）；

- 示例：

  cpp

  

  运行

  

  

  

  

  ```
  String s2("world");
  String s3 = std::move(s2); 
  // s2的str已为nullptr，以下操作无意义/崩溃
  // cout << s2.getStr() << endl; // ❌ 错误：s2已无资源
  ```

  

### 五、实战对比：深拷贝 vs 移动语义（一眼看出效率）

cpp



运行









```
int main() {
    // 场景1：深拷贝（慢）
    String s1("这是一个很长很长的字符串，长度100000字节...");
    String s2 = s1; // 深拷贝：复制100000字节，慢

    // 场景2：移动（快）
    String s3 = std::move(s1); // 移动：仅复制指针，零开销
    return 0;
}
```

**输出结果**：

plaintext











```
普通构造：0x55e8b87aeb20
深拷贝构造：0x55e8b87afb40  // 新内存地址，复制数据
移动构造：0x55e8b87aeb20    // 复用原地址，无复制
析构：释放0x55e8b87aeb20
析构：释放0x55e8b87afb40
析构：释放(nullptr)
```

### 六、核心注意事项（避坑关键）

1. **移动后源对象的状态**：合法但未定义（通常为空），只能析构，不能访问资源；
2. **noexcept 关键字**：移动构造 / 赋值建议加`noexcept`，否则标准容器（如`vector`）可能 fallback 到深拷贝；
3. **标准库已支持移动语义**：`string`、`vector`、`map`等容器都内置了移动构造，直接用`std::move`即可提升性能；
4. **不要滥用 std::move**：对短期存在的对象移动无意义，仅对大对象 / 资源型对象（堆内存、文件句柄）使用。

### 总结（快速掌握的 3 个核心）

1. **核心目的**：用 “资源转移” 替代 “深拷贝”，解决大对象复制的性能问题；
2. **核心语法**：右值引用`&&` + 移动构造函数`类名(类名&&)` + `std::move`（转左值为右值）；
3. **核心规则**：移动构造只偷指针、清空源对象；`std::move`仅做类型转换，移动后源对象不可用。

### 速记口诀

- 左值有名字，右值是临时；
- && 绑右值，移动不拷贝；
- std::move 转类型，偷完资源要清空

### **总结**

移动语义是用来代替大对象传递的时候性能问题，适用于，在从A作用域传递到B作用域的时候，对象在A作用域已经没有用了。
