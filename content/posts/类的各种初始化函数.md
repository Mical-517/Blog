# 一、类的初始化核心：全是「构造函数」

所有构造函数共同点：

- 函数名 = 类名
- 无返回值
- **对象创建时自动调用**
- 唯一目的：给新对象做初始化（赋初值、分配内存、申请资源）

------

# 二、6 种初始化相关构造 / 机制（按考试 + 实际开发优先级）

## 1. 初始化列表

**不是函数，但它是「真正初始化成员」的地方**

- 写法：

  ```
  类名(...) : 成员1(值), 成员2(值) { ... }
  ```

  

- **必须用它**：

  - 初始化 `const` 成员
  - 初始化 **引用成员**
  - 成员是一个**没有默认构造**的类对象

  

- **场景**：所有构造函数都推荐优先用它，比在函数体内赋值更早、效率更高。

------

## 2. 默认构造函数（无参构造）

- **作用**：创建一个 “空对象”，不需要外部传值。

- **触发**：

  

  

  ```
  Student stu;       // 调用默认构造
  Student arr[10];   // 数组必须用默认构造
  ```

  

- **规则**：

  - 你不写任何构造 → 编译器自动生成一个空的默认构造
  - 你写了任意构造 → 编译器**不再生成**

  

- **场景**：需要创建无初始值对象、对象数组、放入容器（vector 等）。

------

## 3. 带参构造函数（普通构造）

- **作用**：创建对象时**直接传入初始属性**。

- 示例：

  

  

  ```
  String s("hello");
  Point p(10, 20);
  ```

  

- **场景**：绝大多数业务对象初始化。

------

## 4. 拷贝构造函数（你刚学的重点）

- **作用**：用 **一个已经存在的对象** 初始化 **新对象**。

- 固定写法：

  

  ```
  类名(const 类名& 其他对象)
  ```

  

- **一定会触发的 3 个场景**：

  1. `Student s2 = s1;`
  2. 函数**值传递**参数 `void func(Student s)`
  3. 函数**值返回**对象 `Student func(){...}`

  

- **关键区分**：

  - 类里**无指针 / 无堆内存** → 用编译器默认的**浅拷贝**就行
  - 类里**有指针 / 管理堆内存** → 必须手写**深拷贝**，否则崩溃

  

------

## 5. 移动构造函数（C++11 优化）

- **作用**：用 **临时对象（将亡值）** 初始化新对象，**偷资源，不复制**。

- 写法：

  ```c++
  类名(类名&& 其他对象)
  ```

  

- **场景**：

  - 函数返回大对象（string、vector、自定义大对象）
  - 临时对象赋值
  - 追求高性能，避免深拷贝浪费时间

  

------

## 6. 委托构造函数

- **作用**：一个构造函数 **调用本类另一个构造函数**，复用代码。

- 示例：

  

  

  ```c++
  Student() : Student(0, "") {}       // 委托带参构造
  Student(int id, string name) {...}
  ```

  

- **场景**：多个构造有重复初始化逻辑，减少冗余代码。

------

# 三、一句话快速判断：该用哪个？

1. 只想创建空对象 → **默认构造**
2. 创建时直接给属性 → **带参构造**
3. 复制一个已有的对象 → **拷贝构造（深 / 浅）**
4. 临时对象拿来用，想快 → **移动构造**
5. 多个构造代码重复 → **委托构造**
6. 只要在构造里初始化成员，优先用 **初始化列表**

------

# 四、总结

1. **拷贝构造必须传 const 引用**，否则无限递归
2. **有指针必写深拷贝**，否则双重 delete 崩溃
3. **初始化列表优先级 > 构造体内赋值**
