# 1.现在电脑开机通电了

首先，os是一个软件，运行就要装在memory，如何载入的呢？计算机有一点固有的引导程序BIOS，就使用来引导os开始运行工作

![image-20260313210515294](https://bucket-qjy.oss-cn-qingdao.aliyuncs.com/picture/202603132105316.png)

## 第一阶段：开始运行bootsect.s

bootsect程序，（一开始cpu处于实时模式，只能寻址16位），首先将自身程序转移到另一个位置，空出当前位置（后面解释），然后将setup程序模块装载到内存，将system模块装载到0地址处，这就是为什么一开始bootsect就要腾出空间，防止覆盖原来的程序。

主要作用：加载setup程序模块。



## 第二阶段：运行setup模块（完成os启动前的设置）



setup模块初始化一些硬件参数比如内存大小，方便后续os初始化时分配内存合理，使用一条高级汇编指令（cpu变为保护模式，32位），然后转移指令开始执行system（位于0地址处）



## 第三阶段：执行system程序

system是有许多模块组合的，这时候，先执行什么就尤为重要。

先执行head.s文件，之后进入main函数进行os的初始化（初始化一切os可掌握的资源情况）**注意这里main函数永远不会退出，很好理解，我们使用pc一定是要断电之后，os才不工作**。

引入内核态（段）与用户态（段）

我们已知程序访问内存都是一段一段访问的，所以在不同的内存段给他划分为不同的状态，**那怎么划分的状态**：记得前面head.s初始化gdt表，表里的每个像的地址（一开始不只有操作系统的地址吗），会将最后的两位来表示什么这段地址是什么状态的

流程：现在执行一条指令（执行程序不就是执行指令），然后在gdt表里可以看到目前这段程序（指令）是处于什么段，然后将这个状态放到currentPL（cpl）里面，然后当这个程序要访问其他内存段的时候），同样根据gdt来查询DPL（目标段的特权值），如果特权比较低，就不能访问，这就是为什么普通程序不能直接反问操作系统内核里的数据



那么，不能·直接访问，还有什么办法可以访问操作系统呢内核(换句话说普通程序使用系统调用进入内核的内存段访问数据，这个系统调用是使用什么办法可以将用户态段的内存访问内核态的内存)

1. 首先认识中断：中断指令是cpu自带的硬指令，他不是操作系统的，**但是操作系统的系统调用可以访问内核段的地址就是因为这个中断指令，比如int x80**

2. 这个中断的实现机理

   #### 步骤 1：用户态执行 `int 0x80` 指令

   用户程序在 3 特权级（用户态）执行这条指令，指令本身的作用是：「触发 0x80 号软中断，请 CPU 切换到内核处理我的请求」。

   #### 步骤 2：CPU 先查 IDT（中断描述符表），找中断处理入口

   - IDT 是 CPU 硬件级的表，记录了「每个中断号（比如 0x80）对应哪个处理程序」；
   - CPU 拿到 0x80 这个中断号，会去 IDT 里找第 0x80 项，这一项里存着「系统调用处理函数的内核地址」，以及「执行这个处理函数需要的特权级（0 级，即内核态）」。

   #### 步骤 3：CPU 硬件校验 + 切换特权级（核心：GDT 出场）

   这是权限切换的关键，完全由 CPU 硬件自动完成，程序无法干预：

   1. CPU 检查 IDT 项里的特权级要求（必须是 0 级），确认当前是从 3 级（用户态）触发，符合 “低特权级调用高特权级” 的规则；

   2. CPU 去 **GDT（全局描述符表）** 里，找到内核代码段、内核数据段的描述符（这些描述符标记了 “仅 0 特权级可访问”）；

   3. CPU 自动修改段寄存器：

      - CS（代码段寄存器）→ 切换到内核代码段；
      - SS（栈段寄存器）→ 切换到内核栈段；
      - 同时把 CPU 的当前特权级 CPL 从 3（用户态）改成 0（内核态）；

      

   4. 保存用户态现场（比如用户态的寄存器值、程序计数器），防止切换回来后程序跑飞。

   #### 步骤 4：执行内核的系统调用处理函数

   此时 CPU 已经是内核态（0 级），能合法访问内核段内存：

   1. 内核从用户态传入的寄存器（比如 eax 寄存器存系统调用号，如 1=exit、3=read），判断用户要执行什么操作；
   2. 调用内核里对应的具体函数（比如 sys_read、sys_write），完成文件读写、内存分配等内核级操作；
   3. 操作完成后，内核把结果写回用户态能访问的寄存器（比如 eax 存返回值）。

   #### 步骤 5：切回用户态

   内核执行 `iret` 指令，CPU 硬件自动：

   1. 恢复之前保存的用户态现场；
   2. 把 CS/SS 切回用户段，CPL 从 0 改回 3；
   3. 回到用户程序中 `int 0x80` 指令的下一行，继续执行用户态代码。
