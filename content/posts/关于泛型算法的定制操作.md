# 可调用对象与function

## 1.可调用对象：

1. 函数
2. 函数指针
3. lamba表达式
4. bind函数创建的对象
5. 重载了()的类

总结：可调用对象，就是可以**可调用对象** 本质是：**任何能通过 `()` 运算符传递参数并执行的实体** —— 不管它是函数、指针、类对象还是匿名表达式，只要能像 `f(args)` 这样调用，就是可调用对象。



2.可调用对象当然也有各自的类型，定义一个lamba就会自动生成一个类类型，函数以及函数指针类型有他的参数以及返回值决定



3.理解调用形式，**调用形式是所有调用对象拥有的一个概念，就是接受什么参数返回什么类型**（就是一种函数类型），有这样的情况就是即使类型不同的可调用对象，可以使用同一种调用形式

```c++
//普通函数
int add(int a,int b) {return i+b;}

//lamba表达式
auto mod=[](int a,int b) ->int (return a%b);

//重载()的类类型
class divide
{
    public:
    int operator(int a,int b) return a/b;
}

//上述三种调用对象，即使他们执行不同操作，是不同类型，但是都使用同一种调用形式

```

实际问题：

现在比如我们想要实现一个桌面计算器的简单实现，想要调用上面的函数，一个简单想法就是有个表，然后表里面放函数指针就，使用什么函数就使用对应的函数指针实现，这个表可以使用map实现，键值就可以是string表示操作名字，**但是存放的value是什么类型呢，如果是   int(*)(int ,int )，显然这是一个函数类型（或者说调用形式），但是显然这个类型与lamba的类类型不匹配，与指向类成员函数指针也不匹配，那该怎么办，怎么统一他们的类型？**

## 2.标准库function类型

function是一种新的标准库类型，定义在functional头文件

function操作

1. function<T> f	f是一个用来存储可调用对象的空function，这些可调用对象的**调用形式**应该与函数类型T相同
2. function<T> f{obj}  在f中存储obj可调用对象的副本
3. f                              作为判断条件，如果有可调用对象为真
4. f(args)                    调用f中的对象，参数是args



**function就是一个模版类型，需要我们提供可调用对象的调用形式** 

```c++                           
function<int(int,int)> f1=add;/divide/[](int a,int b )->int {return a/b;}
f1(a,b);

//基于此我们就可以统一有一个类型来存储不同的可调用对象，实现可调用对象的类型统一化
//实现map，//实现可调用对象表

map<int,function<int(int,int)>> binops={
    {"+",add},
    {"-",std::minus<int>()},
    {"/",divide()},
}

//使用
binops["+"](10,2);
binops["-"](10,2);

```



重载的函数与function

对与重载的函数，因为对与function函数来说它初始化的方法就是接受指针（函数名）或者是一个可调用对象（比如lamba），所以如果函数有重载，他会分不清哪一个重载是满足int(int ,int )这种调用形式的

```c++
int add(int a,int b);
size_t add (size_t a,size_t b);
function<int(int,int)>> f=add;	//使用指针初始化，但是他分不清是哪一个add
```
