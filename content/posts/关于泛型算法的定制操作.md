# 一.可调用对象与function

## 1.可调用对象：

1. 函数
2. 函数指针
3. lamba表达式
4. bind函数创建的对象
5. 重载了()的类

总结：可调用对象，就是可以**可调用对象** 本质是：**任何能通过 `()` 运算符传递参数并执行的实体** —— 不管它是函数、指针、类对象还是匿名表达式，只要能像 `f(args)` 这样调用，就是可调用对象。



2.可调用对象当然也有各自的类型，定义一个lamba就会自动生成一个类类型，函数以及函数指针类型有他的参数以及返回值决定



3.理解调用形式，**调用形式是所有调用对象拥有的一个概念，就是接受什么参数返回什么类型**（就是一种函数类型），有这样的情况就是即使类型不同的可调用对象，可以使用同一种调用形式

```c++
//普通函数
int add(int a,int b) {return i+b;}

//lamba表达式
auto mod=[](int a,int b) ->int (return a%b);

//重载()的类类型
class divide
{
    public:
    int operator(int a,int b) return a/b;
}

//上述三种调用对象，即使他们执行不同操作，是不同类型，但是都使用同一种调用形式

```

实际问题：

现在比如我们想要实现一个桌面计算器的简单实现，想要调用上面的函数，一个简单想法就是有个表，然后表里面放函数指针就，使用什么函数就使用对应的函数指针实现，这个表可以使用map实现，键值就可以是string表示操作名字，**但是存放的value是什么类型呢，如果是   int(*)(int ,int )，显然这是一个函数类型（或者说调用形式），但是显然这个类型与lamba的类类型不匹配，与指向类成员函数指针也不匹配，那该怎么办，怎么统一他们的类型？**

## 2.标准库function类型

function是一种新的标准库类型，定义在functional头文件

function操作

1. function<T> f	f是一个用来存储可调用对象的空function，这些可调用对象的**调用形式**应该与函数类型T相同
2. function<T> f{obj}  在f中存储obj可调用对象的副本
3. f                              作为判断条件，如果有可调用对象为真
4. f(args)                    调用f中的对象，参数是args



**function就是一个模版类型，需要我们提供可调用对象的调用形式，他就是为所有的可调用对象提供另一个统一的接口** 

```c++                           
function<int(int,int)> f1=add;/divide/[](int a,int b )->int {return a/b;}
f1(a,b);

//基于此我们就可以统一有一个类型来存储不同的可调用对象，实现可调用对象的类型统一化
//实现map，//实现可调用对象表

map<int,function<int(int,int)>> binops={
    {"+",add},
    {"-",std::minus<int>()},
    {"/",divide()},
}

//使用
binops["+"](10,2);
binops["-"](10,2);

```



重载的函数与function

对与重载的函数，因为对与function函数来说它初始化的方法就是接受指针（函数名）或者是一个可调用对象（比如lamba），所以如果函数有重载，他会分不清哪一个重载是满足int(int ,int )这种调用形式的

```c++
int add(int a,int b);
size_t add (size_t a,size_t b);
function<int(int,int)>> f=add;	//使用指针初始化，但是他分不清是哪一个add
//解决办法就是,明确给一个指向目的函数的指针
int (*fp)(int ,int)=add;//自动指向正确的add函数
function<int(int,int)> f=fp;//利用指针初始化
```



# 二.定制操作

起源：对与标注库算法，是可以允许我们自定义操作来代替算法内部原本的操作的



## 1.向算法传递函数

对与重载的算法版本，它支持我们传递谓词，这里谓词就是一个**可调用的表达式**，他接受参数，然后返回一个可以作为条件判断的值，谓词可以分为一元谓词和二元谓词，他们表示接收参数的个数。这里当我们传递谓词的时候，算法就会对每一个元素都执行这个谓词，所以参数类型必须与容器内的元素相匹配

```c++
bool isShorter(const string& s1,const string& s2)
{
    return s1.size()<=s2.size();	//小于为真
}

sort(begin,end,isShorter);
```



## 2.lamba表达式

起源：

因为算法对与谓词的类型有严格限制（比如规定一定要是一元还是二元），所以lamba这时候就可以发挥作用，再者对与一些不常用的简单函数完全可以写成lamba表达式来简化代码



### 1.认知lamba表达式

首先lamba表达式本身就是一个可调用对象，他可以接收参数返回一个类型，所以他自己就是一个可调用的代码单元，**可以理解为一个没有命名的内联函数**。

```c++
//lamba的形式
[捕获列表](接受参数列表)->后置返回类型{函数体};
//说明，一个lamba表达式最基本要具有[捕获列表，可以为空]{函数体}

//比如
auto f=[]{return 42};
f()//返回42
    
    
```

### 2.向lamba传递参数

1. 实参类型与形参类型匹配
2. lamba表达式不支持默认实参

### 3.使用捕获列表

**这里捕获列表**，只可以捕获lamba所在作用域已经定义好的变量，捕获之后就意味着lamba在表达式就可以直接使用这个变量，而不用在参数列表声明

```c++
sort(words.begin(),words.end(),isShorter);//先按照字数排序
//我们想要输出所有size>=5的单词
int size=5;
//一个简单想法：
{
    bool isSize(const string& s,int size)
    {
        return s.size()>=size;
    }
    iterator i=find_if(begin,end,isSize);//找到第一个>=size的元素，返回指向它的迭代器
    //这里❌️，因为find_if只接受一个一元谓词，这时候就可以使用lamba
}
{
    iterator i=find_if(begin,end,[size](const string&s ){return s.size()>=size});
    //这里lamba表达式是捕获他所在函数内部的变量，在其函数内部使用，本身还是一个一元谓词
}
    
```



### 4.详细探索lamba捕获以及返回

首先清楚一点，**当我们顶一个lamba**，编译器生成一个lambda对应的新的（未命名）的类类型，所以在默认情况下，但我们定义一个lambda是类内部就会使用捕获的变量进行初始化

#### 值捕获

当我们使用值捕获时，就是进行一个拷贝操作（当然前提是变量可以进行拷贝），但是传递参数是不同，**lambda，被捕获的变量是在lambda定义的时候就进行拷贝，不是在调用时拷贝**

```c++
int a=4;
auto f=[a](){return a};
a=0;
cout<<f()<<endl;	//输出4而非0
```

#### 引用捕获

类比常规引用，就是使用的源对象，不是拷贝的对象。但是我们要确保的是，在lambda使用引用变量的时候，这个变量是存在的，不会被销毁

有时引用捕获是必要的，因为有一些变量是不支持拷贝操作的比如操作流对象

#### 可变lambda

一般来说，当lambda使用值捕获的形式来捕获一个变量来说，函数体内部是不可以修改这个值的，因为在函数体内部认为这是一个不可以修改的左值，但是解决办法就是加上一个mutable

```c++
int v=2;
auto f=[v](){return ++v}//❌️
auto f=[v]()mutable{return ++v}//✅️
cout<<v<<endl;//2
cout<<f()<<endl;//3

```



注意的是：**对与只在一两个地方使用的简单操作使用lambda是最简单的**





### 3.参数绑定

正如我们前面所说的，有些算法会对谓词的参数做出限制，除了lambda使用捕获参数克服这一个困难，还**有一种方法就是使用bind函数**，来克服如何向第二个参数传递参数的问题

#### 标注库的bind函数

**bind函数定义在头文件functional中**，**可以将bind函数看做一个通用的函数适配器**（类比容器适配器，是底层容器的不同，从而展现不同的特性），他接受一个**可调用对象**，生成一个新的可调用对象来适应源对象的参数列表

```c++
//调用bind的一般形式
auto newCallable=bind(callable,arg_list);
//callable是一个可调用对象，就这个生成一个新的可调用对象是newCallable，arg_list与callable的参数对应，当我们调用newCallable时，就会调用callable，并将arg_list传递给callable
```

对与arg_list,我们可以发现参数有的为_n,这里n泛指整数，表示占位符，他意思就是占据了传递给newCAllable的参数的位置_，_1为第一个参数，以此类推_

绑定参数实例

```c++
check_size(string& s,int size);
auto check6=bind(check_size,_1,6);//表示bind调用只需要一个参数，占位符出现在第一个位置，表示check6的此参数对应check_size的第一个参数string&，占位符在std::placeholders::当中

//所以替换lambda
auto i=find_if(begin,end,bind(check_size,_1,size));

```



bind的参数互换

```c++
//比如对与
void printn(int a,int b,int c,int d,int e);
auto f=bind(print,a,b,_2,c,_1);
//这里f的意思就是调用f时将第一个参数（也就是_1)传递给print的最后一个参数
f(_1,_2)---------> printn(a,b,_2,c,_1);

//实例
isshorter(stirng s1,string s2)return s1.size()<s2.size();
sort(begin,end,bind(isshorter(_2,_1)));	//会依照降序排序


```



关于bind绑定引用

```c++
void print(ostream& os,string s);

for_each(beign,end,bind(print)(os,_1));//错，os不能够复制
for_each(beign,end,bind(print)(ref(os),_1));
```
