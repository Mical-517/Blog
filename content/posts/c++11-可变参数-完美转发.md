# 一.问题

在 C++98 中：

1. 函数 / 模板只能写**固定个数**的参数
2. 无法优雅地传递「任意参数」给构造函数
3. 无法保留参数的**左值 / 右值属性**（导致拷贝浪费性能）

所以 C++11 推出这套特性，只为解决一个问题：

**让函数能接收「任意个数、任意类型、任意值属性」的参数，并原封不动转发给目标函数（比如构造函数）**



# 二.具体解决

### 1.可变参数模版 class... Args

1. 定义

```c++
template <class T, class... Args>
```

- `class... Args`：**模板参数包**
- `...` 是「包声明符」，表示：**接收 0 ~ 任意多个 类型参数**
- `Args` 不是一个类型，是**一组类型的集合**（比如 `int, string, double`）

2. 配套：函数参数包 `Args&&... args`

```c++
void construct(T* ptr, Args&&... args);
```

- `Args&&... args`：**函数参数包**
- 对应接收：**0 ~ 任意多个 函数实参**

3. 举个例子（ `construct`）

```c++
construct(p, 10, 3.14, "hello");
```

- 模板参数包 `Args`  →  `<int, double, const char*>`
- 函数参数包 `args`  →  `10, 3.14, "hello"`

4. 参数包展开 `...`

这是可变参数的**核心用法**：`...` 放在参数包后面，**编译期自动展开所有参数**

```c++
// 你的代码
mystl::forward<Args>(args)...

// 编译期自动展开为：
forward<int>(10), forward<double>(3.14), forward<const char*>("hello")
```

✅ 无运行时开销，纯编译期操作



### 2.万能引用Args&&

`Args&&` ≠ 右值引用！

规则（死记硬背）：

- **普通代码**中的 `&&` = 右值引用（只能绑定临时对象）

  

  ```c++
  void fun(int&& a) {} // 纯右值引用
  ```

- **模板推导 + T&&** 中的 `&&` = **万能引用**（可绑定任意值）

  

  ```c++
  template <class T>
  void fun(T&& a) {} // 万能引用！
  ```

万能引用的能力

它能接收 **3 种参数**，通吃所有：

1. 左值（有名字的变量：`int a = 10`）
2. 右值（临时值：`10`, `string("abc")`）
3. 任意类型（int/string/ 自定义类）



### 3.引用折叠（完美转发，万能引用的底层原理）

编译器遇到「多层引用叠加」时，会自动合并，这就是**引用折叠**

T&  &   →  T&  （左值引用） 

T&  &&  →  T&  （左值引用） 

T&& &   →  T&  （左值引用）

 T&& &&  →  T&& （右值引用）

**只有「右值 + 右值」才会折叠成右值引用，其余全是左值引用**



### 4.完美转发

作用只有一个：

 **原封不动保留参数的「值属性」**

- 传左值 → 转发成左值（调用拷贝构造）
- 传右值 → 转发成右值（调用移动构造，高效）

为什么需要保留：

如果不用，万能引用会**丢失右值属性**，所有参数都会变成左值，无法调用移动构造，性能暴跌。

假设你写一个**普通版本的 construct**，不用 `Args&&`，只用普通传参：

```c++
// 普通版本，没有万能引用
template<class T, class Arg>
void construct(T* ptr, Arg arg)
{
    // arg 现在有名字！
    // 不管你传进来是左值还是右值，arg 都是左值
    ::new (ptr) T(arg); 
}
```

现在你传一个**右值临时对象**进去：

```c++
string s = "hello";

// 传了一个右值（临时string）
construct(p, string("world")); 
```

### 发生了什么？

1. 你传进去的 `string("world")` 是**纯右值**，本来可以**移动**
2. 但函数里用 `Arg arg` 接收，`arg` 是**有名字的变量**
3. 编译器看到 `arg` 是左值 → **只能调用 string 的拷贝构造**
4. **右值属性彻底丢失**



# 总结

串联：你的 `construct` 函数 = 四大特性合体

这是 TinySTL 核心代码，**一行行对应四大特性**：

```c++
// 1. class... Args   → 可变参数模板（接收任意类型）
template<class T, class... Args>
// 2. Args&&... args  → 万能引用（接收任意值）
void construct(T* ptr, Args&&... args)
{
    // 3. forward      → 完美转发（保留值属性）
    // 4. ...          → 参数包展开（编译期展开所有参数）
    ::new ((void*)ptr) T(mystl::forward<Args>(args)...);
}
```

## 它能做到什么？（STL 容器的核心能力）

1. 支持**无参构造**

   

   ```c++
   construct(p)
   ```

2. 支持**拷贝构造**

   ```c++
   construct(p, 10);
   ```

3. 支持**移动构造**

   ```c++
   construct(p, move(10));
   ```

4. 支持**多参构造**（哪怕类没有默认构造函数）

   ```c++
   construct(p, "Tom", 18);
   ```
