# 数据库基础认识

## 1. 数据库为什么产生？

数据库是**为了解决早期数据管理方式的严重缺陷**而出现的：

1. **人工管理**：数据不保存、不共享、和程序绑死，完全无法处理大量数据。
2. **文件系统**：数据散在不同文件里，出现**冗余多、不一致、难共享、数据和程序耦合太紧**等问题。

为了**统一、高效、安全、共享地管理大量数据**，实现数据独立、减少冗余、保证一致，数据库技术才诞生。

------

## 2. 数据库到底是什么？

**数据库（DB，Database）**：

长期存放在计算机内，**有组织、可共享、统一管理**的相关数据的集合。

大白话：

不是零散的表格 / 文件，而是一个**按规则整理好、能被多人共用、方便增删改查的数据仓库**。

## 3. 数据库由什么组成

1. 数据库本身：他是信息的载体，**在细分的话，每个数据库都由若干个表组成，每个表有表头等信息（类似excel表格）**，表里面存储一条条信息
2. 数据库管理系统：他是管理数据库的软件系统，他可以对数据库进行操作，实现信息的增删改查
3. 基本客户端：提供界面使用命令来调用数据库管理系统进行对信息的操作

# 数据库基础篇

## 1. 数据库通用语法规范

1. 不区分大小写
2. 可以使用空格分隔
3. 使用分号表示一条命令结束
4. **字段注释使用单引号（更推荐）**



## 2. 基本的命令

### 1. DDL命令(数据定义语句)

#### 1. 对库进行操作

```mysql
//查询所有数据库
show databases; //注意复数
//查询操纵的当前数据库
select database();

//创建数据库
create database [if not exists] 数据库名称 [default charset 字符集][collate 排序规则]

//删除数据库
drop database [if exists] 数据库名称
	
//使用数据库
use 数据库名称

```



#### 2. 对表进行操作

```mysql
//查询当前库中的所有表
show tables;

//查询表结构（desc：description描述）
desc 表名称;

//查询建表所有语句
show create table 表名称;

//创建表
create table 表名称(
字段1 字段一类型 comment '注释',
字段2 字段二类型 comment '注释',
字段3 字段三类型 comment '注释'    //最后一个字段不需要逗号
) comment '表注释'


//修改表（格式：指定表+操作）
//添加字段
alter table 表名字 add 字段名 字段类型 comment '注释';

//修改已有字段类型
alter table 表名字 modify 字段名 新类型;

//整体修改字段
alter table 表名 change 旧字段 新字段名 数据类型 comment '注释';

//删除字段
alter table 表名 drop 字段名;


//删除表
drop table [if exists] 表名;

//格式化表
truncate table 表名;

```

### 2.DML(数据控制语句)

```mysql
//1. 增加一条信息
insert into 表名 (字段1,字段2，····) values(值1，值2，·····); //这是指定为信息的字段1，字段2赋值

insert into 表名 values(值1，值2，值3······)；//这条信息的所有字段都要进行赋值
/*
注意：1.值与字段一一对应
	 2.varchar,时间日期都要使用单引号括起来
*/


//修改一条信息
update 表名 set 指定字段=某个值 where 所要修改信息的标识   //比如如果有字段id，写id=1就是所有id=1，执行前面的赋值操作，省略where语句就是，将所有的信息，都执行update操作



//删除一条信息
delete from 表名 where 指定信息     //同样如果没有where就删除所有信息
```



### 3.DQL(数据查询语句)

#### 1.基本查询

```
//查询对应字段列表
select 字段1,字段2 from 表名

//查询所有字段
select * from 表名

//查询字段时，把显示的字段替换为别名
select 字段 as 别名 from 表名

//查询相应的字段是，如果这一列有数据一样只显示一次，数据去重
select distinct 字段 from 表名


```

#### 2.条件查询

```
//基本语法
select 字段列表 from 表名 where 条件

//条件分为	比较运算和逻辑运算

```

#### 3.聚合函数

聚合函数时针对一个或者多个字段（就是按照列为单位）进行操作,常见操作：`COUNT()`（计数）、`SUM()`（求和）、`AVG()`（平均值）、`MAX()`（最大值）、`MIN()`（最小值）。

```mysql
-- 创建示例表
CREATE TABLE student_score (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_name VARCHAR(50),
    subject VARCHAR(20),
    score INT,
    exam_date DATE
);

-- 插入测试数据
INSERT INTO student_score (student_name, subject, score, exam_date) VALUES
('张三', '数学', 90, '2026-01-10'),
('张三', '语文', 85, '2026-01-10'),
('李四', '数学', 88, '2026-01-10'),
('李四', '语文', 92, '2026-01-10'),
('王五', '数学', 75, '2026-01-10'),
('王五', '语文', 80, '2026-01-10'),
('张三', '数学', 95, '2026-02-15'), -- 张三第二次数学考试
('李四', '语文', 89, '2026-02-15');


-- 1. COUNT()：统计记录总数（* 代表所有行，也可以用具体字段）
SELECT COUNT(*) AS total_records -- 统计所有行数
FROM student_score; -- 结果：8

SELECT COUNT(score) AS valid_score_count -- 统计score字段非NULL的行数（本例中无NULL，结果也是8）
FROM student_score;

-- 2. SUM()：计算某个字段的总和
SELECT SUM(score) AS total_score -- 所有分数的总和
FROM student_score; -- 结果：90+85+88+92+75+80+95+89 = 694

-- 3. AVG()：计算某个字段的平均值
SELECT AVG(score) AS avg_score -- 所有分数的平均分（保留2位小数）
FROM student_score; -- 结果：694/8 = 86.75

-- 4. MAX()/MIN()：找字段的最大值/最小值
SELECT MAX(score) AS max_score, MIN(score) AS min_score
FROM student_score; -- 结果：max=95，min=75
```
