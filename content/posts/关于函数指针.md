# 函数指针

## 1.普通函数指针

声明：

```c++
using funcPtr=int(*)(string);	//表示funcPtr是一个指针类型，指向的是一个接受string参数，返回int的函数
```

定义

```c++
int func(string){}
funcPtr ptr=&func;	//函数名就是函数地址 //❌️
funPtr ptr=func;	//不需要&;

或者是
int(*ptr)=&func; 	//❌️,不需要&
int(*ptr)
```

调用

```c++
ptr("test");
```



## 2.成员函数指针

声明：

```c++
using memberfunPtr=int(类名::*)(string);
```

定义

```c++
memberfunPtr ptr=&类名::func;	//需要&
```

调用

```c++
(类对象.(*ptr))(string);
```

## **关于成员函数的混淆点**

**一定要理解，成员函数属于类，类中的成员变量才属于类对象**，这就意味着不同类对象调用相同函数，呈现不同的效果是因为他们各自的成员变量不同，与成员函数无关，成员函数是共有的。

在成员函数指针上的体现：就比如第70行与第74行，我一开始认为会调用myclass1中的函数，输出王二，但是还是输出李四

原因：成员函数是共有的，所以看似将myclass.ptr = myclass1.ptr;，**还是指向同一块内存**，**还是看是哪个具体对象调用的才打印对应对象的名字**

### 总结

1. **成员函数指针指向 “类的函数”**：而非某个对象的函数，所有对象的同名成员函数指针指向的是同一个函数地址。
2. **调用时的对象决定 this 指针**：`(对象.*指针)(参数)`中，“对象” 决定使用哪个对象的成员变量（如`name`），指针仅决定调用哪个成员函数。
3. **修改指针的意义**：只有让指针指向**不同的成员函数**（如`func1`/`func2`），才能看到调用结果的差异；指向同一个函数的不同对象指针，修改后无效果。
